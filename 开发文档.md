# 🌈 趣学堂 - 儿童学习网站开发文档

## 一、项目概述

### 1.1 项目名称
**趣学堂**（FunLearn）

### 1.2 项目定位
面向 4-7 岁学龄前及低年级儿童的趣味学习平台，涵盖拼音、汉字、英语、数学及课外拓展知识，通过游戏化学习方式激发学习兴趣，帮助儿童在轻松愉快的氛围中掌握基础知识。

### 1.3 核心理念
- **玩中学**：每个知识点都融入游戏机制，让学习像玩耍一样自然
- **短时高效**：单次学习 5-10 分钟，避免注意力疲劳
- **螺旋上升**：知识由浅入深，已学内容通过游戏反复巩固
- **正向激励**：星星、徽章、成就系统持续激励孩子前进
- **家长可控**：家长可查看学习报告、设置学习时长和内容

### 1.4 项目目标
| 目标 | 描述 |
|------|------|
| 知识掌握 | 拼音全部声母韵母、100+ 常用汉字、50+ 英语单词、20 以内加减法 |
| 学习兴趣 | 通过游戏化设计让孩子主动要求学习 |
| 记忆巩固 | 间隔重复算法确保知识长期记忆 |
| 习惯养成 | 每日学习打卡，培养良好学习习惯 |

## 二、目标用户分析

### 2.1 主要用户：5 岁左右儿童

| 特征 | 设计应对 |
|------|----------|
| 注意力持续时间短（5-10 分钟） | 单次学习模块控制在 5 分钟内 |
| 识字量少，无法阅读说明 | 全程语音引导，图标化操作 |
| 手指精细动作尚在发展 | 大按钮、大触摸区域，减少拖拽操作 |
| 喜欢鲜艳色彩和动画 | 丰富动画反馈，卡通风格 UI |
| 需要即时正反馈 | 每次操作都有音效和视觉奖励 |
| 容易受挫 | 无惩罚机制，错误时温和鼓励 |

### 2.2 次要用户：家长

| 需求 | 设计应对 |
|------|----------|
| 了解学习进度 | 学习报告面板（正确率、学习时长、掌握情况） |
| 控制学习时间 | 可设置每日学习时长上限 |
| 选择学习内容 | 可自定义学习计划和内容范围 |
| 多孩家庭 | 支持多用户档案切换 |

### 3.2 拼音王国

#### 3.2.1 学习内容
| 阶段 | 内容 | 知识点数量 |
|------|------|-----------|
| 第一阶段 | 声母（b p m f d t n l g k h j q x zh ch sh r z c s y w） | 23 个 |
| 第二阶段 | 单韵母（a o e i u ü） | 6 个 |
| 第三阶段 | 复韵母（ai ei ui ao ou iu ie üe er） | 9 个 |
| 第四阶段 | 鼻韵母（an en in un ün ang eng ing ong） | 9 个 |
| 第五阶段 | 整体认读音节（zhi chi shi ri zi ci si yi wu yu ye yue yuan yin yun ying） | 16 个 |
| 第六阶段 | 声调与拼读 | 4 个声调 + 拼读规则 |

#### 3.2.2 学习流程（每个拼音 3-5 分钟）
```
听音认形 → 跟读练习 → 书写练习 → 小游戏巩固
   ↓           ↓           ↓           ↓
 播放发音    录音对比    描红书写    拼音消消乐
 展示口型    评分反馈    笔顺动画    听音选图
 关联图片    反复练习    自由书写    拼音配对
```

#### 3.2.3 互动设计
- **发音动画**：每个拼音配有口型动画和发音示范
- **跟读评分**：语音识别技术，对孩子发音进行星级评分（1-3 星）
- **描红书写**：触摸屏/鼠标描红，笔顺引导动画
- **记忆卡片**：每个拼音配一张趣味图片（如：a → 阿姨的嘴型图）

#### 3.2.4 巩固游戏
| 游戏 | 玩法 | 巩固目标 |
|------|------|----------|
| 拼音消消乐 | 点击相同拼音配对消除 | 拼音认读 |
| 听音选图 | 听发音选择对应图片 | 音形关联 |
| 拼音射击 | 气球飘出拼音，点击正确的气球 | 快速识别 |
| 拼读小火车 | 声母韵母组合拼读，拖拽车厢 | 拼读能力 |
| 声调小船 | 根据声调划船到正确码头 | 声调辨别 |

#### 3.3.2 学习流程（每个汉字 2-3 分钟）
```
看图识字 → 笔顺演示 → 书写练习 → 组词拓展 → 游戏巩固
   ↓           ↓           ↓           ↓           ↓
 图片+汉字    动态笔顺    描红+自由    2-3个组词    识字游戏
 发音播放    笔画拆解    评分反馈    简单造句     反复出现
 字源故事    书空练习    连笔练习    关联记忆     间隔复习
```

#### 3.3.3 特色功能
- **字源动画**：展示汉字从甲骨文到楷书的演变过程（如"日"从太阳图案演变而来）
- **笔顺动画**：逐笔展示书写顺序，可慢放、回放
- **组词连线**：将汉字与对应图片连线
- **造句填空**：简单句式填空，如"___在天上飞"（鸟）

#### 3.3.4 巩固游戏
| 游戏 | 玩法 | 巩固目标 |
|------|------|----------|
| 汉字钓鱼 | 钓起与图片匹配的汉字鱼 | 字义关联 |
| 拼字积木 | 偏旁+部件组合拼字 | 字形结构 |
| 汉字消消乐 | 找出相同汉字配对消除 | 字形识别 |
| 识字跑酷 | 角色奔跑，选择正确汉字通过障碍 | 快速认读 |
| 词语开花 | 中心字向外扩展组词 | 组词能力 |

### 3.5 数学城堡

#### 3.5.1 学习内容
| 阶段 | 内容 | 知识点 |
|------|------|--------|
| 第一阶段 | 1-10 数字认知 | 认数、数数、数字书写 |
| 第二阶段 | 1-20 数字认知 | 认数、数数、比较大小 |
| 第三阶段 | 10 以内加法 | 逐步从实物过渡到抽象 |
| 第四阶段 | 10 以内减法 | 逐步从实物过渡到抽象 |
| 第五阶段 | 20 以内加法 | 不进位→进位 |
| 第六阶段 | 20 以内减法 | 不退位→退位 |
| 第七阶段 | 趣味逻辑 | 规律发现、简单排序、分类 |

#### 3.5.2 学习流程
```
实物演示 → 动画理解 → 互动操作 → 练习巩固 → 游戏应用
   ↓           ↓           ↓           ↓           ↓
 苹果/糖果   动画演示    拖拽操作    限时答题     数学游戏
 数量变化    过程可视化  实物计数    逐步提升     生活应用
 直观感受    抽象过渡    独立计算    即时反馈     逻辑思维
```

#### 3.5.3 教学策略
- **从实物到抽象**：先用苹果、糖果等实物演示，再过渡到数字符号
- **分步引导**：3+2=? → 先数3个苹果，再数2个苹果，合在一起数
- **多种表征**：数字、实物图、数轴、手指计数多种方式展示
- **错误分析**：常见错误类型提示（如进位忘记加1）

### 3.7 游戏中心

#### 3.7.1 每日挑战
- 每天推送一组综合题目（5-8 题），涵盖已学内容
- 根据学习进度智能出题，薄弱知识点优先
- 完成获得"每日之星"奖励

#### 3.7.2 复习闯关
- 基于间隔重复算法（Spaced Repetition），自动安排复习
- 闯关模式：每关 5 题，全部答对进入下一关
- 关卡难度递增，从单科到综合

#### 3.7.3 知识竞技
- 限时答题模式，与虚拟角色 PK
- 题目类型随机，涵盖所有学科
- 排行榜激励（仅展示自己的最佳成绩，避免攀比）

#### 3.7.4 自由探索
- 开放式学习地图，孩子自由选择感兴趣的内容
- 隐藏彩蛋和惊喜，鼓励探索
- 收集系统：探索发现新知识获得收集品

## 四、游戏化设计体系

### 4.1 奖励机制

```
即时奖励（每次操作）
  ├── 正确：星星飞出动画 + 欢快音效 + "+1星"
  ├── 连续正确：连击特效 + 额外星星
  └── 错误：温和鼓励 "再试一次" + 提示引导

阶段奖励（完成一个学习单元）
  ├── 完成学习：获得 5-10 颗星星
  ├── 全部正确：获得"完美通关"徽章
  └── 首次完成：获得"初次尝试"徽章

每日奖励（每日学习目标）
  ├── 完成每日挑战：获得宝箱（含星星+随机道具）
  ├── 连续打卡：第7天双倍奖励，第30天特殊奖励
  └── 每日首次学习：获得"早起鸟"徽章

成就奖励（长期目标）
  ├── 里程碑成就：解锁特殊角色皮肤
  ├── 学科成就：解锁对应学科高级游戏
  └── 全科成就：解锁"小学者"称号
```

### 4.2 防挫败机制
- **无惩罚**：答错不扣分、不扣星星
- **渐进提示**：第一次错给提示，第二次错给答案+讲解
- **难度自适应**：连续答错自动降低难度
- **鼓励语**：随机鼓励（"你真棒！""加油哦！""差一点点！"）
- **保底奖励**：即使全错也获得参与星星

### 4.3 间隔重复算法

采用改良版 SM-2 算法，适配儿童学习节奏：

```
复习间隔：
  第1次复习：学习后 1 天
  第2次复习：上次复习后 2 天
  第3次复习：上次复习后 4 天
  第4次复习：上次复习后 7 天
  第5次复习：上次复习后 15 天

评分标准（简化为3档）：
  😊 完全正确 → 间隔延长
  🤔 有犹豫/提示后正确 → 间隔不变
  😕 错误 → 间隔缩短，1天后重新复习
```

## 五、学习时长控制设计

### 5.1 时间策略

| 场景 | 时长 | 说明 |
|------|------|------|
| 单个知识点学习 | 2-3 分钟 | 认识→练习→巩固 |
| 单个学习单元 | 5-8 分钟 | 3-4 个知识点 |
| 单次学习总时长 | 15-20 分钟 | 2-3 个学习单元 |
| 每日学习总时长 | 20-30 分钟 | 分 2-3 次完成 |
| 游戏复习 | 5-10 分钟 | 独立于学习时长 |

### 5.2 时间控制机制

```
学习开始
  │
  ├── 5分钟 → 弹出提示："你已经学习5分钟了，真棒！继续加油！"
  │
  ├── 10分钟 → 弹出提示："休息一下吧！站起来活动活动～"
  │            └── 提供1分钟眼保健操动画
  │
  ├── 15分钟 → 弹出提示："今天学了很多了，去玩个游戏复习一下吧！"
  │            └── 自动跳转到游戏中心
  │
  └── 20分钟（或家长设置的上限）→ 强制休息
               └── "学习时间到啦！休息一会儿再来吧～"
               └── 锁定学习内容，仅保留游戏中心
               └── 5分钟后可继续（或等到下一个时段）
```

### 5.3 注意力保持策略
- 每 2-3 分钟切换活动类型（看→听→做→玩）
- 动画和音效变化保持新鲜感
- 即时反馈避免等待无聊
- 小目标拆分，每完成一个小目标有奖励

## 六、UI/UX 设计规范

### 6.1 视觉风格
- **整体风格**：扁平化 + 微拟物，温暖可爱的卡通风格
- **主色调**：暖色系为主
  - 主色：#FF9F43（温暖橙）
  - 辅色：#54A0FF（天空蓝）
  - 强调色：#FF6B6B（活力红）
  - 背景色：#FFF9E6（暖米白）
  - 成功色：#2ED573（清新绿）
- **字体**：圆体/幼圆风格，字号偏大（最小 18px）
- **图标**：圆润线条图标，线条粗（3-4px）

### 6.2 交互规范
| 规范 | 说明 |
|------|------|
| 按钮最小尺寸 | 48px × 48px（触摸设备 64px × 64px） |
| 按钮间距 | 至少 16px |
| 操作反馈 | 所有操作 200ms 内给出视觉/听觉反馈 |
| 页面切换 | 使用趣味过渡动画（如：云朵飘过、门打开） |
| 导航层级 | 最多 3 层，任何页面 2 步可回到首页 |
| 加载等待 | 使用趣味动画（如：小动物跑步、星星旋转） |

### 6.3 页面布局原则
- **顶部**：返回按钮 + 当前位置 + 星星数量
- **中部**：主要内容区域（占屏幕 70%）
- **底部**：导航栏（首页、学习、游戏、我的）
- **安全区域**：重要操作按钮避开屏幕边缘

### 6.4 动画规范
- 页面进入：300ms 弹入动画
- 按钮点击：100ms 缩放反馈
- 正确反馈：星星飞出 + 欢快音效（500ms）
- 错误反馈：轻微摇晃 + 鼓励音效（300ms）
- 页面切换：500ms 过渡动画

### 6.5 音效规范
| 场景 | 音效 |
|------|------|
| 按钮点击 | 清脆"叮"声 |
| 正确答案 | 欢快上升音阶 |
| 错误答案 | 温和"嗯嗯"声（非刺耳） |
| 获得星星 | "叮铃铃"声 |
| 完成学习 | 胜利音乐 |
| 提醒休息 | 轻柔提示音 |
| 背景音乐 | 轻柔欢快的循环音乐（可关闭） |

```prisma
model User {
  id          String    @id @default(cuid())
  name        String
  avatar      String
  birthDate   DateTime?
  parentId    String    @map("parent_id")
  parent      Parent    @relation(fields: [parentId], references: [id])
  stars       Int       @default(0)
  level       Int       @default(1)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  learningRecords LearningRecord[]
  achievements   UserAchievement[]
  inventory      UserInventory[]
  dailyLogs      DailyLog[]
  reviewQueue    ReviewItem[]
}

model Parent {
  id        String  @id @default(cuid())
  username  String  @unique
  password  String
  nickname  String
  children  User[]

  settings  ParentSettings?
}

model ParentSettings {
  id                String  @id @default(cuid())
  parentId          String  @unique @map("parent_id")
  dailyTimeLimit    Int     @default(30)
  sessionTimeLimit  Int     @default(10)
  restInterval      Int     @default(15)
  allowedStartTime  String  @default("08:00")
  allowedEndTime    String  @default("21:00")
  enabledModules    String  @default("pinyin,chinese,english,math,explore")
  parent            Parent  @relation(fields: [parentId], references: [id])
}

model LearningRecord {
  id          String   @id @default(cuid())
  userId      String   @map("user_id")
  user        User     @relation(fields: [userId], references: [id])
  module      String   // pinyin, chinese, english, math, explore
  topic       String   // 具体知识点，如 "声母-b", "汉字-日"
  action      String   // learn, practice, review, game
  score       Int?     // 正确率 0-100
  starsEarned Int      @default(0)
  duration    Int      // 学习时长（秒）
  createdAt   DateTime @default(now())
}

model Achievement {
  id          String  @id @default(cuid())
  code        String  @unique
  name        String
  description String
  icon        String
  category    String
}

model UserAchievement {
  id            String    @id @default(cuid())
  userId        String    @map("user_id")
  achievementId String    @map("achievement_id")
  user          User      @relation(fields: [userId], references: [id])
  achievement   Achievement @relation(fields: [achievementId], references: [id])
  unlockedAt    DateTime  @default(now())

  @@unique([userId, achievementId])
}

model ReviewItem {
  id          String   @id @default(cuid())
  userId      String   @map("user_id")
  user        User     @relation(fields: [userId], references: [id])
  module      String
  topic       String
  level       Int      @default(0)    // SM-2 记忆等级
  nextReview  DateTime // 下次复习时间
  lastReview  DateTime?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

model DailyLog {
  id              String   @id @default(cuid())
  userId          String   @map("user_id")
  user            User     @relation(fields: [userId], references: [id])
  date            DateTime @db("Date")
  totalDuration   Int      @default(0)
  starsEarned     Int      @default(0)
  itemsCompleted  Int      @default(0)
  streak          Int      @default(0)
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  @@unique([userId, date])
}

model UserInventory {
  id          String   @id @default(cuid())

}
```

## 十一、关键设计决策

### 11.1 为什么选择 Vue 3 + Phaser？
- Vue 3 的组合式 API 适合模块化开发学习组件
- Phaser 是成熟的 2D 游戏引擎，适合开发教育类小游戏
- 两者可以很好地集成，Vue 管理页面，Phaser 管理游戏画布

### 11.2 为什么采用本地优先策略？
- 5 岁儿童使用场景主要在家庭，网络不稳定时也能学习
- 减少服务器依赖，降低运维成本
- 学习数据本地缓存，网络恢复后同步

### 11.3 为什么不用现成的教育平台？
- 个性化：可根据孩子进度定制内容
- 隐私保护：数据自控，不依赖第三方
- 可扩展：随时添加新内容和新游戏
- 成本低：无需持续付费订阅
